# Vlinder Chat - Copilot API 迁移到 Vlinders API 计划

**项目**: Vlinder Chat VSCode Extension
**目标**: 将 GitHub Copilot API 迁移到 Vlinders 自有服务平台
**产品网站**: https://vlinders.org/
**计划日期**: 2026-02-28
**版本**: v1.0

---

## 📋 执行摘要

### 迁移目标
1. ✅ **保留 GitHub 认证** - 用户可继续使用 GitHub 账号登录并管理仓库
2. ❌ **移除 Copilot 订阅依赖** - 不再依赖 GitHub Copilot 付费服务
3. ✨ **构建 Vlinders API** - 复刻 Copilot API 架构，建立自有 AI 服务平台
4. 💰 **支持订阅服务** - 为未来商业化做准备

### 保留功能
- GitHub OAuth 认证
- GitHub 仓库管理（PR、Issues、代码浏览）
- VSCode 扩展 UI 和交互
- Agent 系统和工具调用
- 网络请求层（FetcherService）
- 端点管理系统（EndpointProvider）

### 新增功能
- Vlinders 账号系统（可选，与 GitHub 账号并存）
- Vlinders API Key 管理
- 订阅计划管理
- 使用量统计和配额控制

## 📦 Vlinders API 功能规划

### 核心 API 端点

#### 1. 认证与授权
```
POST   /v1/auth/login          # 用户登录
POST   /v1/auth/refresh        # 刷新 Token
GET    /v1/auth/me             # 获取当前用户信息
DELETE /v1/auth/logout         # 登出
```

#### 2. 聊天与对话
```
POST   /v1/chat/completions    # OpenAI 兼容格式
POST   /v1/responses           # Copilot 响应格式
POST   /v1/messages            # Anthropic Messages 格式
```

#### 3. 代码补全
```
POST   /v1/completions         # 代码补全建议
POST   /v1/completions/stream  # 流式补全
```

#### 4. 文本嵌入
```
POST   /v1/embeddings          # 生成文本嵌入向量
```

#### 5. Agent 工具调用
```
POST   /v1/agents/execute      # 执行 Agent 任务
GET    /v1/agents/tools        # 获取可用工具列表
```

#### 6. 模型管理
```
GET    /v1/models              # 获取可用模型列表
GET    /v1/models/{id}         # 获取模型详情
```

#### 7. 订阅与计费
```
GET    /v1/subscription        # 获取订阅信息
GET    /v1/usage               # 获取使用量统计
GET    /v1/quota               # 获取配额信息
```

### API 响应格式

#### 聊天响应（流式）
```typescript
// SSE 事件流
data: {"id":"msg_001","type":"content_block_start","content_block":{"type":"text","text":""}}
data: {"id":"msg_001","type":"content_block_delta","delta":{"type":"text_delta","text":"Hello"}}
data: {"id":"msg_001","type":"content_block_stop"}
data: {"id":"msg_001","type":"message_stop","usage":{"input_tokens":10,"output_tokens":5}}
data: [DONE]
```

## 🔐 认证方案设计

### 双认证系统

#### GitHub 认证（保留）
- **用途**: 仓库管理、PR、Issues
- **流程**: OAuth 2.0
- **作用域**: `read:user`, `repo`, `workflow`
- **存储**: VSCode Secret Storage

#### Vlinders 认证（新增）
- **用途**: AI 服务调用
- **流程**: API Key 或 OAuth 2.0
- **Token 格式**: JWT
- **存储**: VSCode Secret Storage

### 认证流程

```
用户启动扩展
  ↓
检查 GitHub Session (用于仓库功能)
  ↓
检查 Vlinders Token (用于 AI 功能)
  ↓
如果 Vlinders Token 不存在或过期
  ↓
提示用户登录 Vlinders
  ↓
选项 1: 使用 GitHub 账号关联登录
选项 2: 使用 Vlinders 账号登录
选项 3: 使用 API Key
  ↓
获取 Vlinders Token
  ↓
缓存 Token (有效期 24 小时)
  ↓
开始使用 AI 功能
```

## ⚠️ 风险和挑战

### 技术风险

1. **API 兼容性**
   - 风险: Vlinders API 与 Copilot API 格式不完全兼容
   - 缓解: 在客户端做适配层

2. **性能问题**
   - 风险: 自建服务性能不如 GitHub Copilot
   - 缓解: 使用 CDN、缓存、负载均衡

3. **认证复杂度**
   - 风险: 双认证系统增加复杂度
   - 缓解: 清晰的认证流程设计和文档

### 业务风险

1. **用户迁移**
   - 风险: 现有用户不愿意切换
   - 缓解: 提供免费试用期、平滑迁移

2. **服务稳定性**
   - 风险: 初期服务不稳定
   - 缓解: 充分测试、灰度发布
